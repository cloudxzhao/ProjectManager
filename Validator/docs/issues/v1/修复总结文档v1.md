## 已修复问题清单

### 1. BACKEND-002: 用户更新 profile 时 nickname 覆盖 username（严重）

**问题描述**：UserController 的 updateProfile 方法接受 nickname 参数，但实际上错误地将其设置为 username 字段，导致用户无法使用原用户名登录。

**修复内容**：
- 在 `User.java` 实体添加 `nickname` 字段
- 修改 `UserService.updateProfile()` 方法，将 nickname 参数正确设置到 nickname 字段
- 在 `UserVO.java` 添加 nickname 字段
- 添加数据库迁移脚本 `V4__add_user_nickname.sql`

**修改文件**：
- `src/main/java/com/projecthub/module/user/entity/User.java`
- `src/main/java/com/projecthub/module/user/service/UserService.java`
- `src/main/java/com/projecthub/module/user/dto/UserVO.java`
- `src/main/resources/db/migration/V4__add_user_nickname.sql`

---

### 2. BACKEND-004: 资源不存在返回 500 而非 404

**问题描述**：当请求的资源不存在时，后端返回 HTTP 500 状态码而不是 404。

**修复内容**：
- 修改 `ErrorCode.java` 枚举，添加 httpStatus 字段
- 修改 `BusinessException.java`，支持 httpStatus 属性
- 修改 `GlobalExceptionHandler.java`，根据异常返回正确的 HTTP 状态码
- 修改 `ProjectService.java`、`TaskService.java` 中的异常抛出，使用 ErrorCode 枚举

**修改文件**：
- `src/main/java/com/projecthub/common/constant/ErrorCode.java`
- `src/main/java/com/projecthub/common/exception/BusinessException.java`
- `src/main/java/com/projecthub/common/exception/GlobalExceptionHandler.java`
- `src/main/java/com/projecthub/module/project/service/ProjectService.java`
- `src/main/java/com/projecthub/module/task/service/TaskService.java`

---

### 3. BACKEND-005: 未认证返回 403 而非 401

**问题描述**：当用户未提供 token 或 token 无效时访问受保护接口，应返回 401 而不是 403。

**修复内容**：
- 在 `SecurityConfig.java` 中添加 `authenticationEntryPoint` 和 `accessDeniedHandler`
- 未认证返回 401 Unauthorized
- 无权限返回 403 Forbidden

**修改文件**：
- `src/main/java/com/projecthub/common/config/SecurityConfig.java`

---

### 4. BACKEND-008: Token 刷新返回 500 错误

**问题描述**：使用 refreshToken 刷新访问令牌时返回 500 错误。

**修复内容**：
- 修改 `AuthService.refreshToken()` 方法，改进异常处理
- 使用 `ErrorCode.TOKEN_INVALID` 返回正确的错误码和状态码
- 区分业务异常和系统异常

**修改文件**：
- `src/main/java/com/projecthub/module/auth/service/AuthService.java`

---

### 5. BACKEND-009: UserVO 缺少 nickname 字段

**问题描述**：UserVO 中没有定义 nickname 字段。

**修复内容**：
- 在 `UserVO.java` 中添加 nickname 字段

**修改文件**：
- `src/main/java/com/projecthub/module/user/dto/UserVO.java`

## 数据库迁移

## 修复验证步骤

1. **运行数据库迁移**
   ```bash
   # 确保 PostgreSQL 容器运行
   docker ps | grep postgres

   # 应用 Flyway 迁移（应用启动时自动执行）
   mvn spring-boot:run -Dspring-boot.run.profiles=dev
   ```

2. **验证修复**
   ```bash
   # 1. 登录获取 token
   TOKEN=$(curl -s -X POST http://localhost:9527/api/v1/auth/login \
     -H "Content-Type: application/json" \
     -d '{"usernameOrEmail":"admin","password":"Admin123"}' | jq -r '.data.accessToken')

   # 2. 测试未认证返回 401
   curl -s -o /dev/null -w "%{http_code}" http://localhost:9527/api/v1/user/profile
   # 期望：401

   # 3. 测试无效 token 返回 401
   curl -s -o /dev/null -w "%{http_code}" http://localhost:9527/api/v1/user/profile \
     -H "Authorization: Bearer invalid_token"
   # 期望：401

   # 4. 测试获取不存在的项目返回 404
   curl -s -o /dev/null -w "%{http_code}" http://localhost:9527/api/v1/projects/9999 \
     -H "Authorization: Bearer $TOKEN"
   # 期望：404

   # 5. 测试更新用户资料（nickname）
   curl -X PUT "http://localhost:9527/api/v1/user/profile?nickname=TestUser" \
     -H "Authorization: Bearer $TOKEN"
   # 期望：nickname 被正确保存

   # 6. 测试 Token 刷新
   REFRESH_TOKEN=$(curl -s -X POST http://localhost:9527/api/v1/auth/login \
     -H "Content-Type: application/json" \
     -d '{"usernameOrEmail":"admin","password":"Admin123"}' | jq -r '.data.refreshToken')

   curl -X POST http://localhost:9527/api/v1/auth/refresh \
     -H "Content-Type: application/json" \
     -d "{\"refreshToken\":\"$REFRESH_TOKEN\"}"
   # 期望：返回新的 token
   ```
