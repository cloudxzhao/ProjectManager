# 前后端 API 接口一致性校验任务

## 任务目标
对比后端实际 API 与前端期望 API 的差异，识别接口不一致问题，确保前后端对接正确。

## 数据源

| 来源 | 类型 | 获取方式 |
|------|------|----------|
| 后端 API | OpenAPI 3.0.1 JSON | `curl -s http://localhost:8080/v3/api-docs` |
| 前端 API | OpenAPI 3.0.3 YAML | `/data/project/ProjectManager/Builder/frontend/docs/api/API 前端接口文档.md` |

## 对比维度

### 1. URI 差异对比
- **路径一致性**：检查前端调用的路径是否在后端存在对应接口
- **路径参数命名**：如 `{id}` vs `{taskId}` 是否一致
- **HTTP 方法**：GET/POST/PUT/DELETE 是否匹配
- **缺失接口**：前端调用但后端不存在的接口
- **多余接口**：后端存在但前端未调用的接口

### 2. 入参差异对比
| 参数类型 | 检查项 |
|----------|--------|
| Path 参数 | 参数名、类型、是否必填 |
| Query 参数 | 参数名、类型、默认值、是否必填 |
| Request Body | 字段名、字段类型、嵌套结构、必填项 |

### 3. 响应差异对比
| 检查项 | 说明 |
|--------|------|
| HTTP 状态码 | 200/400/401/403/404/500 等是否一致 |
| 响应结构 | `Result<T>` 包装结构字段是否一致 |
| data 类型 | 返回的具体数据对象结构是否一致 |
| 分页结构 | `PageResult<T>` 字段是否一致 |

### 3. 风险评估
- 🔴 **高风险**：会导致功能完全不可用
- 🟡 **中风险**：可能导致部分功能异常
- 🟢 **低风险**：仅影响文档一致性，不影响功能

- 需要更新前端文档的接口

## 注意事项
1. 重点关注 CRUD 操作的接口一致性
2. 注意 `projectId` 路径参数是否存在差异
3. 检查分页参数 `page`/`size` 的默认值是否一致
4. 验证 `required` 字段的定义是否一致
