## 发现的问题

### 1. 字段命名不一致

**问题描述**：后端返回的项目数据中，颜色字段使用 `themeColor`，而前端使用 `color`。

**后端定义**（ProjectVO.java）：
```java
@JsonProperty("color")
private String themeColor;
```

**解决方案**：
- 在前端 API 客户端 (`src/lib/api/project.ts`) 中添加数据映射函数 `mapProjectResponse`
- 将后端的 `themeColor` 字段映射为前端的 `color`

```typescript
const mapProjectResponse = (response: ProjectResponse): Project => ({
  ...response,
  color: response.themeColor, // 后端 themeColor -> 前端 color
});
```

---

### 2. 日期字段类型

**问题描述**：后端的 `startDate` 和 `endDate` 使用 `LocalDate` 类型，格式化后为 `yyyy-MM-dd` 字符串。

**后端定义**：
```java
@JsonFormat(pattern = "yyyy-MM-dd")
private LocalDate startDate;
```

**前端处理**：
- 前端类型定义使用 `string` 类型接收
- 如果需要显示格式化，可在组件中处理

---

### 3. 分页参数名称不一致

**问题描述**：
- 前端期望：`page`, `pageSize`
- 后端接收：`page`, `size`

**后端定义**（ProjectController.java）：
```java
@RequestParam(defaultValue = "1") Integer page,
@RequestParam(defaultValue = "20") Integer size
```

**解决方案**：前端调用时使用 `size` 而不是 `pageSize`

```typescript
const params = new URLSearchParams();
params.append('page', page.toString());
params.append('size', pageSize.toString()); // 使用 size
```

---

### 4. 分页响应字段不一致

**问题描述**：
- 前端期望：`items`, `total`, `page`, `pageSize`, `totalPages`
- 后端返回：`list`, `total`, `page`, `size`, `pages`

**后端定义**（PageResult.java）：
```java
private List<T> list;
private Long total;
private Integer page;
private Integer size;
private Integer pages;
```

**解决方案**：前端调整类型定义以匹配后端

```typescript
return api.get<{
  list: ProjectResponse[];  // 不是 items
  total: number;
  page: number;
  size: number;
  pages: number;  // 不是 totalPages
}>(url, { params });
```

---

### 5. Axios 类型转换问题

**问题描述**：`axios.ts` 中的类型转换导致 TypeScript 编译错误。

**原始代码**：
```typescript
get: <T>(url: string, config?: AxiosRequestConfig) =>
  apiClient.get<Result<T>>(url, config) as Promise<Result<T>>,
```

**错误信息**：
```
Type 'AxiosResponse<Result<T>, any, {}>' is missing the following properties from type 'Result<T>': code, message, timestamp
```

**解决方案**：使用 `.then()` 提取 `response.data`

```typescript
get: <T>(url: string, config?: AxiosRequestConfig) =>
  apiClient.get<Result<T>>(url, config).then((res) => res.data),
```

---

### 6. 项目状态枚举

**问题描述**：后端使用字符串表示项目状态，需要确保前端类型定义一致。

**后端状态值**：`planning`, `active`, `completed`, `archived`

**前端类型定义**：
```typescript
export type ProjectStatus = 'planning' | 'active' | 'completed' | 'archived';
```

**解决**：保持前后端状态值一致
