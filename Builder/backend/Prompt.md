## 1. WorkTree 协作Prompt  

背景：我们有一个基于 Spring Boot 的后端项目（ProjectHub 项目管理平台），采用标准的 Maven 多模块架构。

  现状：
  - 项目采用单体仓库（Monorepo）管理
  - 多个后端工程师需要并行开发不同功能模块
  - 希望在避免代码冲突的前提下，提升协作效率

  需求：
  请基于 Git Worktree 特性，设计一套多工作区开发方案，内容包括：

  1. 工作区划分策略
     - 按什么维度划分工作区最合适？（按功能模块/按开发人员/按特性分支）
     - 推荐的工作区目录结构

  2. Git 分支管理策略
     - 主分支、开发分支、特性分支的命名规范
     - 各工作区对应的分支checkout策略

  3. 实际操作指南
     - 创建/删除工作区的命令示例
     - 多工作区开发的日常操作流程

  4. 冲突预防与合并策略
     - 如何识别和避免模块间代码冲突
     - 推荐的代码合并与同步流程

  5. 注意事项与最佳实践
     - 多工作区开发的常见陷阱
     - 团队协作规范建议
