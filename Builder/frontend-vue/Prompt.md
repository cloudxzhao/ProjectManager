## 1. WorkTree 划分方案的 Prompt

 ### 需求背景

  我们有一个 Next.js 14 + React 18 + TypeScript 的前端项目，采用以下架构：

  - **路由**: Next.js App Router (`src/app/`)
  - **组件**: 按功能模块组织 (`src/components/features/`)
  - **状态管理**: Zustand + TanStack Query
  - **UI 库**: Ant Design 5.x

  ### 团队情况

  - 前端团队：4-6 人
  - 每人负责不同功能模块（如：认证、项目管理、任务看板、报表等）
  - 所有人同时开发，需要频繁提交代码

  ### 核心诉求

  使用 Git Worktree 实现：

  1. **并行开发**：每人独立工作区，互不干扰
  2. **减少冲突**：不同模块的代码尽量不冲突
  3. **易于集成**：定期合并到主分支，冲突可控
  4. **环境独立**：每人可独立运行、调试、测试

  ### 请提供

  1. **Worktree 划分方案**：按什么维度创建工作区（按人？按模块？）
  2. **分支策略**：每个 worktree 对应什么分支，分支如何命名
  3. **目录映射**：每个 worktree 负责哪些具体目录/文件
  4. **工作流程**：从创建 worktree 到合并代码的完整流程
  5. **冲突预防**：识别潜在冲突点，给出预防措施
  6. **命令参考**：具体的 git worktree 操作命令
