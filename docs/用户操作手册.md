## 1. 系统概述

### 1.1 简介

ProjectHub 是一款现代化的项目管理系统，旨在帮助团队高效协作、管理项目和追踪任务进度。系统提供了完整的项目生命周期管理功能，包括项目创建、任务分配、进度追踪、文档协作等核心功能。

### 1.2 核心功能

| 功能模块 | 描述 |
|---------|------|
| 项目管理 | 创建、编辑、归档项目，管理项目成员和权限 |
| 任务看板 | 可视化任务管理，支持拖拽、状态流转、子任务 |
| 用户故事 | 史诗(Epic)和用户故事(User Story)管理 |
| 问题追踪 | Bug 和 Issue 的创建、分配、追踪和解决 |
| Wiki 文档 | 团队知识库，支持版本历史 |
| 通知中心 | 实时通知，任务更新提醒 |

### 1.3 系统要求

- **浏览器**: Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- **网络**: 稳定的网络连接
- **分辨率**: 建议 1920×1080 或更高

## 2. 快速开始

### 2.1 访问系统

1. 打开浏览器，访问系统地址
2. 进入登录页面

### 2.2 注册账户

1. 点击登录页面的「注册」链接
2. 填写注册信息：
   - **用户名**: 唯一标识，用于登录
   - **邮箱**: 用于密码找回和通知
   - **密码**: 至少 8 位，包含字母和数字
3. 点击「注册」按钮
4. 注册成功后自动跳转到登录页面

### 2.3 登录系统

1. 输入用户名和密码
2. 点击「登录」按钮
3. 登录成功后跳转到仪表盘页面

## 3. 用户账户管理

### 3.1 忘记密码

1. 在登录页面点击「忘记密码」
2. 输入注册邮箱
3. 系统发送重置邮件
4. 点击邮件中的链接设置新密码

### 3.2 修改密码

1. 进入「设置」→「安全设置」
2. 输入当前密码和新密码
3. 点击「确认修改」

### 3.3 个人资料

1. 进入「设置」页面
2. 修改个人信息：
   - 显示名称
   - 个人简介
   - 联系方式
3. 上传头像
4. 点击「保存更改」

## 4. 项目管理

### 4.1 创建项目

1. 点击顶部导航栏的「项目」
2. 点击「新建项目」按钮
3. 填写项目信息：

   | 字段 | 说明 | 必填 |
   |------|------|------|
   | 项目名称 | 项目标题 | 是 |
   | 描述 | 项目简介 | 否 |
   | 状态 | 规划中/进行中/已完成/已归档 | 是 |
   | 开始日期 | 项目开始时间 | 否 |
   | 结束日期 | 项目结束时间 | 否 |

4. 点击「创建」完成创建

### 4.2 查看项目列表

- 访问「项目」页面查看所有项目
- 支持按状态筛选：规划中、进行中、已完成、已归档
- 支持搜索项目名称
- 项目卡片显示：名称、状态、成员数量、任务统计

### 4.3 项目详情

点击项目卡片进入项目详情页，可查看：

- **概览**: 项目基本信息、统计数据
- **任务**: 项目下的所有任务
- **成员**: 项目成员列表
- **用户故事**: 关联的用户故事

### 4.4 项目成员管理

#### 添加成员

1. 进入项目详情页 →「成员」标签
2. 点击「添加成员」
3. 搜索用户（按用户名或邮箱）
4. 选择角色：
   - **管理员 (ADMIN)**: 完全管理权限
   - **经理 (MANAGER)**: 可管理任务和成员
   - **成员 (MEMBER)**: 只能查看和执行任务
5. 点击「确认添加」

#### 移除成员

1. 在成员列表找到目标成员
2. 点击「移除」按钮
3. 确认移除操作

#### 更改角色

1. 在成员列表找到目标成员
2. 点击角色下拉框
3. 选择新角色

### 4.5 编辑项目

1. 进入项目详情页
2. 点击「编辑」按钮
3. 修改项目信息
4. 点击「保存」

### 4.6 删除项目

1. 进入项目详情页
2. 点击「删除」按钮
3. 输入项目名称确认删除
4. 删除操作不可恢复，请谨慎操作

## 5. 任务管理

### 5.1 任务看板

任务看板采用 Kanban 风格，将任务分为四个状态列：

| 状态 | 描述 |
|------|------|
| 待办 (TODO) | 新创建的任务 |
| 进行中 (IN_PROGRESS) | 正在处理的任务 |
| 审核中 (IN_REVIEW) | 等待审核的任务 |
| 已完成 (DONE) | 已完成的任务 |

### 5.2 创建任务

1. 进入项目详情 →「任务」标签
2. 点击「新建任务」或在看板列中点击「+」
3. 填写任务信息：

   | 字段 | 说明 |
   |------|------|
   | 标题 | 任务名称 |
   | 描述 | 任务详细说明 |
   | 优先级 | 低/中/高/紧急 |
   | 负责人 | 指派成员 |
   | 截止日期 | 任务截止时间 |

4. 点击「创建」

### 5.3 任务详情

点击任务卡片查看详情，包含：

- 任务基本信息
- 任务描述
- 子任务列表
- 评论讨论
- 活动日志

### 5.4 移动任务

- **拖拽**: 直接拖动任务卡片到目标状态列
- **菜单**: 点击任务卡片 → 选择「移动到」→ 选择目标状态

### 5.5 子任务管理

1. 在任务详情页找到「子任务」区域
2. 点击「添加子任务」
3. 输入子任务标题
4. 勾选完成子任务

### 5.6 任务评论

1. 在任务详情页找到「评论」区域
2. 输入评论内容
3. 点击「发送」
4. 支持 @提及其他成员

### 5.7 全局任务搜索

1. 访问「任务」页面
2. 可搜索所有项目中的任务
3. 支持按状态、优先级、负责人筛选

## 6. 用户故事

### 6.1 史诗 (Epic)

史诗是一组相关用户故事的集合，代表较大的功能模块或需求。

#### 创建史诗

1. 进入「用户故事」页面
2. 点击「新建史诗」
3. 填写史诗信息：
   - 名称
   - 描述
   - 关联项目

### 6.2 用户故事 (User Story)

#### 创建用户故事

1. 在史诗下点击「添加用户故事」
2. 填写用户故事：
   - **作为**: 用户角色
   - **我想要**: 期望功能
   - **以便于**: 目的/价值
   - **验收标准**: 完成条件
   - **优先级**: 故事优先级
   - **故事点**: 预估工作量

#### 用户故事状态

- 待办
- 进行中
- 已完成

## 7. 问题追踪

### 7.1 问题类型

| 类型 | 描述 |
|------|------|
| BUG | 程序错误或缺陷 |
| ISSUE | 功能问题或需求问题 |
| IMPROVEMENT | 改进建议 |
| TECH_DEBT | 技术债务 |

### 7.2 优先级和严重程度

**优先级**:
- 低 (LOW)
- 中 (MEDIUM)
- 高 (HIGH)
- 紧急 (URGENT)

**严重程度**:
- 轻微 (MINOR)
- 一般 (MAJOR)
- 严重 (CRITICAL)
- 致命 (BLOCKER)

### 7.3 创建问题

1. 进入项目详情 →「问题」标签
2. 点击「新建问题」
3. 填写问题信息：

   | 字段 | 说明 |
   |------|------|
   | 标题 | 问题简述 |
   | 类型 | Bug/Issue/改进/技术债务 |
   | 优先级 | 紧急程度 |
   | 严重程度 | 影响范围 |
   | 描述 | 详细说明 |
   | 指派给 | 负责人 |
   | 环境 | 问题发生环境 |

### 7.4 问题状态流转

```
新建 → 确认 → 进行中 → 已解决 → 已关闭
                    ↓
                重新打开
```

### 7.5 问题处理

1. 查看问题详情
2. 更新状态
3. 添加评论
4. 关联任务
5. 标记解决

## 8. Wiki 文档

### 8.1 创建文档

1. 进入项目详情 →「Wiki」标签
2. 点击「新建文档」
3. 输入文档标题和内容
4. 支持 Markdown 格式
5. 点击「保存」

### 8.2 编辑文档

1. 点击文档进入查看页面
2. 点击「编辑」按钮
3. 修改内容后保存

### 8.3 版本历史

1. 在文档页面点击「历史」
2. 查看所有版本记录
3. 可对比版本差异
4. 可恢复到历史版本

### 8.4 文档目录

- 支持多级目录结构
- 可拖拽调整顺序
- 支持搜索文档

## 9. 通知中心

### 9.1 查看通知

1. 点击顶部导航栏的「消息」图标
2. 查看所有通知列表

### 9.2 通知类型

| 类型 | 描述 |
|------|------|
| 任务分配 | 有新任务指派给你 |
| 任务更新 | 负责的任务状态变更 |

### 9.3 标记已读

- 点击通知自动标记为已读
- 点击「全部已读」批量标记

### 常见问题

**Q: 如何恢复已删除的项目？**

A: 项目删除后无法恢复，请谨慎操作删除功能。

**Q: 任务可以分配给多个人吗？**

A: 目前每个任务只能有一个负责人，但可以通过子任务分配给不同成员。

**Q: 如何导出数据？**

A: 系统暂不支持数据导出功能，后续版本会添加。
