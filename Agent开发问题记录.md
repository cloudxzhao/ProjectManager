## 多终端单Agent开发流程问题总结
1. 前端按照页面开发，没有对 HTML 文件中的 CSS 样式解析，只保留了 HTML5。—— 在前端开发完之后，再次解析CSS样式，补充开发目标。
2. 前后端开发中经常会由于系统环境问题中断 —— 使用 User 级别的 Memory 配置系统环境信息。
3. 前后端都没有完全遵守API契约文件，都与契约文件有所出入，导致前后端联调的问题很多。 —— 通过测试进行前后端的API对比，生成 Issue 单跟踪解决。
4. Validator 在测试过程中发现问题，开始修复BUG导致后端服务不可用，要约定其不能动代码。
4. 前后端测试出问题后，就不能分开修复了，这样会造成BUG永远无法修复成功. —— 到项目目录下让将任务拆小之后进行逐个问题修复，测试，闭环。
