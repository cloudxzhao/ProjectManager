## ADDED Requirements

### Requirement: 数据库拆分策略
系统 SHALL 按服务边界拆分数据库，每个服务拥有独立数据库实例。

#### Scenario: 服务数据库隔离
- **WHEN** 确定服务边界后
- **THEN** 每个服务使用独立的数据库实例，不共享表

#### Scenario: 跨服务数据访问
- **WHEN** 服务 A 需要服务 B 的数据
- **THEN** 系统通过 API 调用获取，禁止直接访问其他服务数据库

#### Scenario: 数据库命名规范
- **WHEN** 创建服务数据库
- **THEN** 数据库名称格式为 `{service_name}_db`，如 `auth_db`、`task_db`

### Requirement: 双写双读过渡
系统 SHALL 支持新旧数据库并行运行的双写双读过渡期。

#### Scenario: 双写模式开启
- **WHEN** 数据迁移进入过渡期
- **THEN** 系统同时写入旧数据库和新数据库

#### Scenario: 数据一致性校验
- **WHEN** 双写运行超过配置时间（如 1 小时）
- **THEN** 系统自动比对新旧数据库数据差异

#### Scenario: 双读模式切换
- **WHEN** 新数据库数据验证通过
- **THEN** 系统切换读取源为新数据库，旧数据库作为备份

#### Scenario: 双写关闭
- **WHEN** 新数据库稳定运行超过配置时间（如 24 小时）
- **THEN** 系统停止写入旧数据库

### Requirement: 数据迁移工具
系统 SHALL 提供数据迁移工具支持数据搬迁。

#### Scenario: 全量数据导出
- **WHEN** 执行数据迁移命令
- **THEN** 系统从旧数据库导出全量数据到中间文件

#### Scenario: 数据转换与清洗
- **WHEN** 数据导出后
- **THEN** 系统按新表结构转换数据格式，清洗无效数据

#### Scenario: 数据导入验证
- **WHEN** 数据导入新数据库
- **THEN** 系统验证记录数量、关键字段一致性

#### Scenario: 增量数据同步
- **WHEN** 全量迁移完成但未切换
- **THEN** 系统持续同步增量变更到新数据库

### Requirement: 数据回滚机制
系统 SHALL 支持数据迁移失败时的回滚。

#### Scenario: 迁移失败回滚
- **WHEN** 数据迁移过程中发生错误
- **THEN** 系统回滚到迁移前状态，保留原始数据

#### Scenario: 数据库快照备份
- **WHEN** 开始数据迁移前
- **THEN** 系统自动创建源数据库快照备份

#### Scenario: 回滚验证
- **WHEN** 执行回滚操作后
- **THEN** 系统验证数据完整性并输出报告

### Requirement: 数据校验报告
系统 SHALL 生成数据迁移校验报告。

#### Scenario: 迁移前报告
- **WHEN** 数据迁移开始前
- **THEN** 系统生成源数据统计报告（表数量、记录数、存储大小）

#### Scenario: 迁移后报告
- **WHEN** 数据迁移完成后
- **THEN** 系统生成对比报告，列出差异项

#### Scenario: 校验失败告警
- **WHEN** 数据校验发现不一致
- **THEN** 系统触发告警并暂停后续步骤

### Requirement: 外键关系处理
系统 SHALL 正确处理跨表外键关系的迁移。

#### Scenario: 外键依赖分析
- **WHEN** 迁移前分析阶段
- **THEN** 系统识别所有外键关系并生成迁移顺序

#### Scenario: 循环依赖处理
- **WHEN** 存在循环外键依赖
- **THEN** 系统分阶段迁移，先迁移数据后重建约束

#### Scenario: 跨服务关联处理
- **WHEN** 外键指向其他服务的表
- **THEN** 系统将外键转换为 ID 引用，通过 API 维护关联
